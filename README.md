# サイゼリヤシフト管理システムの試作


## 作成の目的
サイゼリヤの毎月のシフト申請が、全て紙媒体で行われている。

このことについて、かなりの不便さを感じ、自身の経験値にもなりうるため、システム開発を私の店長に提案し、着手を決意した。

## 紙媒体のシフト申請の課題点
- 体調不良やインターンシップの参加等で、長期間シフトが入らず、申請期限までに申請できない人がいること。また、今該当者がいなくてもいずれ起こりうる課題であること。

- 記入ミスがあった際に二重線で消すなどして、結果としてわかりづらくなってしまうことが多々あること。

- また、紙でのシフト申請内容や、曜日ごとの勤務時間の変更内容(学生は頻繁に時間が変わりやすい点)を、店長が手作業でデータを入力し、これにより、シフトの組間違い等が起こること。

- 自身が何日に休暇申請をしたのか確実に把握するには、紙媒体での申請では写真を撮る必要があり、「休み申請をした気でいる」状態が発生する可能性があること。

しかし、このシステム実装されたら、上記の問題点を全て解決できると考えている。


## 実装予定内容
  
- 申請画面は、現在の紙媒体と同様のレイアウトの予定
  
  上下左右にスクロールしても、日付・従業員名は常に表示され、自身がどこを見ている常に把握出来るようにする。
- 画面の拡大縮小可能

- 土日祝日を青や赤に

- 申請登録の入力期限設定

  入力はその月の1か月前の頭まで。

  (ex)6月の申請は5/4までなど。

  管理者は月末まで入力可能な設定もいずれ実装しても良いかもしれない。

  表示は、月が変わるまで。
  
  (ex)6月の申請内容は6月末まで閲覧可。
