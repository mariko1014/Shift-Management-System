## 作成の目的
私のバイト先の、毎月のシフト申請は今、全て紙で行われています。

このことについて、かなりの不便さを感じ、自身の経験にも繋がるため、システム開発を店長に提案の上、管理者側での不便な点も聞き取りを行い、着手を決意しました。

### 紙媒体のシフト申請の課題点
- 体調不良やインターンシップの参加等で、長期間シフトが入らず、申請期限までに申請できない人がいること。また、今該当者がいなくてもいずれ起こりうる課題であること。

- 記入ミスがあった際に二重線で消すなどして、結果としてわかりづらくなってしまうことが多々あること。

- また、紙でのシフト申請内容や、曜日ごとの勤務時間の変更内容(学生は頻繁に時間が変わりやすい点)を、店長が手作業でデータを入力しており、これにより、シフトの組み間違い等が起こること。

- 自身が何日に休暇申請をしたのか確実に把握するには、紙での申請では写真を撮る必要があり、「休み申請をした気でいる」状態が発生する可能性があること。

しかし、このシステム実装されたら、上記の問題点を全て解決できると考えています。


## 実装予定内容
  
- 申請画面は、現在の紙のものと同様のレイアウトの予定
  
  ただし、上下左右にスクロールしても、日付・従業員名は常に表示され、自身がどこを見ている常に把握出来るようにします。
- 画面の拡大縮小可能

- 土日祝日を青や赤に

- 申請登録の入力期限設定

  入力/内容編集はその月の1か月前の頭まで。

  (ex)6月の申請は5/4までなど。ただし、期限設定は店長など管理者権限付与者は変更可能に。

  表示は、月が変わるまで。
  
  (ex)6月の申請内容は6月末まで閲覧可。

- 自分の箇所だけ編集できる機能(難しい可能性があります。)

- 管理者権限を持つ人のみ、臨時で入力欄を増やすことができるシステム
  
   新人を雇い入れた際に、急遽枠を増やす。その際、従業員コードと氏名の入力を求め、後々正式にデータベースに登録されたら、コード氏名の照合をすることで欄が二重なる事を防ぐ目的です。(難しい場合は実装先送り)

管理者は制限等なく入力可能な設定もいずれ実装する可能性があります。
