## 作成の目的
私のバイト先にて、毎月のシフト申請は全て紙で行われています。

再来月の勤務日の申請表が、月の頭に出され、記入には店舗に実際に行く必要があります。

(ex)7月の勤務日の申請表が5月頭に店舗休憩室に掲載され、6月頭までに記入する。

このことについて、かなりの不便さを感じ、自身の経験にも繋がるため、システム開発を店長に提案の上、管理者側での不便な点も聞き取りを行い、着手を決意しました。

### 紙媒体のシフト申請の課題点
- 体調不良やインターンシップの参加等で、長期間勤務できず、申請期限までに申請できない人がいること。また、今該当者がいなくてもいずれ起こりうる課題であること。

- 記入ミスがあった際に二重線で消すなどして、結果としてわかりづらくなってしまうことが多々あること。

- また、紙でのシフト申請内容や、曜日ごとの勤務時間の変更内容(学生は頻繁に時間が変わりやすい点)を、店長が手作業でデータを入力しており、これにより、シフトの組み間違い等が発生すること。

- 自身が何日に休暇申請をしたのかを確実に把握するには、紙での申請では写真を撮る必要があり、「休み申請をした気でいる」状態が発生する可能性があること。

しかし、このシステム実装されたら、上記の問題点を全て解決できると考えています。

## 実装予定機能

### 最優先実装
- 新規登録とアカウント情報の保持
- ログイン
- シフト提出
- 申請内容閲覧

### 準優先実装
- 管理者権限
- 申請期限の設定
  
  基本自動設定で、管理者のみ変更可能
- 縦軸横軸の表示固定UI
- Excel出力

### 追加実装予定
- 自身の箇所のみ、入力・編集が可能

- 入力データをExcelに出力し、管理者はExcelで申請内容を確認できる。

- 新規従業員用の欄を臨時で追加可能(管理者のみ)

  その際、従業員コードと氏名の入力を求めます。後々正式にデータベースに登録されたら、コード氏名の照合をすることで欄が二重なる事を防ぐ目的です。


### 実装にあたり意識する事
  
- 申請画面は、現在の紙のものと同様のレイアウトの予定

  ➤フォーマットが変わると、これまで勤めていた従業員にとって使いにくさを感じる可能性があるため

- 土日祝日を青や赤など目立たせる
