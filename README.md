# シフト申請業務の効率化を目的とした管理システム

## 企業の方々へ

本システムは現在個人開発中のため、機能・UIともに順次実装中です。
実装状況に応じて、スクリーンショットやデモサイトURLを追加予定です。

READMEでは、現時点で想定している課題解決や機能設計について記載しています。

## 作成の目的
私のバイト先では、毎月のシフト申請は全て紙で行われています。

再来月の勤務日の申請表が、月の頭に出され、記入には店舗に実際に行く必要があります。

(ex)7月の勤務日の申請表が5月頭に店舗休憩室に掲載され、6月頭までに記入する。

このことについて、かなりの不便さを感じ、自身の経験にも繋がるため、システム開発を店長に提案の上、管理者側での不便な点も聞き取りを行い、着手を決意しました。

### 紙媒体のシフト申請の課題点

- 紙媒体のため、店舗へ直接行かなければ申請できない
- 修正時に二重線で書き直し等を行い、視認性が低下する
- 管理者が紙の情報を手入力でシステムに入力しており、入力ミスが発生する
- 自身の申請内容を後から確認しづらい

しかし、このシステム実装されたら、上記の問題点を全て解決できると考えています。

### 準優先実装

  基本自動設定で、管理者のみ変更可能
- 縦軸横軸の表示固定UI

  その際、出力後の表で、未入力者を強調表示します。

### 追加実装予定
- 直近の申請期限を表示

- 新規登録時に、同じログインIDのユーザーが既に登録されていないかの確認を入れる。

- 新規従業員用の欄を臨時で追加可能(管理者のみ)

  その際、従業員コードと氏名の入力を求めます。後々正式にデータベースに登録されたら、コード氏名の照合をすることで欄が二重なる事を防ぐ目的です。

- シフト申請期限の二段階管理

  通常の申請期限とは別に、急な予定変更へ対応するための
  特別変更期限を設ける予定です。

  ①通常申請期限（月単位）
  管理者が設定した期限まで、申請・編集が可能

  ②特別変更期限（週単位・自動計算）
  通常申請期限超過後～対象週の「2週間前の水曜」まで変更可能

  この期間に変更があった場合は、
  管理者へ通知を送信し、後から変更された申請として判別可能にする

  完全締切後は閲覧のみ可能


### 実装にあたり意識する事
  
- 管理者が確認する画面は、現在の紙のものと同様のレイアウトの予定

  ➤フォーマットが変わると、使いにくさを感じる可能性があるため

- 従業員が申請する画面は月・日・曜日のみを表示し、他従業員の申請内容は表示しないレイアウトにする

  ➤他従業員の欄を選択してしまう等の操作ミスを防ぐため

- 土日祝日を青や赤など目立たせる
